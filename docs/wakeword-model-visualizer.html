Architecture

1. Audio Features
16 × 96
openWakeWord first turns the raw waveform into a compact feature grid. This is what the classifier actually consumes.
2. Flatten
1536 values
The 16 by 96 feature grid is flattened into one long vector so the dense layers can operate on it.
3. Hidden Layer 1
1536 → 32
The network compresses all those feature values into a 32-number internal summary, then applies LayerNorm and ReLU.
4. Hidden Layer 2
32 → 32
A second fully connected block refines that summary into a more wake-word-aware internal representation.
5. Output Score
32 → 1 → sigmoid
The final layer emits one logit, and sigmoid converts it into the confidence score you see in your live tester.

What The Terms Mean

Input feature shape

The model does not see raw audio samples directly. It sees a 16 × 96 table of learned audio features.

Hidden width

The size of each internal dense layer. In your case, each hidden representation contains 32 numbers.

LayerNorm

Keeps the hidden activations in a more stable range while training and inference run.

ReLU

Clips negative activations to zero and keeps positive ones, which helps the network learn non-linear patterns.

Sigmoid

Turns the final raw output into a score between 0 and 1, like 0.12, 0.48, or 0.93.
